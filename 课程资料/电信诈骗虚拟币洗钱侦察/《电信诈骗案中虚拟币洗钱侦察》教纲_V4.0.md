3. **掌握**境内商户配合模式的规范化识别方法（资金匹配度分析、购销材料验证）
4. **掌握**证据调取和保存的规范化操作（截图命名/交易哈希记录表/证据链构建）

## 五、教学内容

### 第1课时：电信网络诈骗案件中虚拟货币洗钱手法与担保交易流程

#### 1.1 三种常见虚拟货币回兑手法（理论+实操 20分钟）

**⚠️ 2024年规范化修订版：专业洗钱团伙+商户配合模式**

**（1）账户出租型虚拟货币回兑（专业洗钱团伙作案模式）** ⭐2024年主流手法

- **核心规范化修正**：账户出租人仅提供账户，**专业洗钱团伙操控人员实际操作所有资金划转**
- 规范化流程：被害人资金划转至账户出租人银行账户 → 账户出租人仅提供账户 → **专业洗钱团伙操控人员实际操作** → 通过OTC渠道购入虚拟货币 → 资金划入**境内正常经营商户银行账户**
- 商户辩解："境外客户通过地下钱庄付款"，能提供购销合同/发货记录等材料
- 规范化侦查突破口：
  - **专业洗钱团伙操控人员**是核心侦查对象（而非账户出租人）
  - 境内商户收款时间与被害人转账时间匹配度分析
  - 资金时间匹配公式：匹配度 ≥ 80% → 高度可疑

**（2）线下现金交付型虚拟货币回兑**

- 规范化流程：被害人提取现金 → 交付指定人员 → 对方现场购入虚拟货币并划转
- 侦查突破口：取款监控录像、手机信令定位、现金交付地点排查
- 手法特点：可切断银行流水证据链；但存在线下监控和人员指认风险

**（3）线下实物购买型虚拟货币回兑**

- 规范化流程：被害人购买黄金/奢侈品等实物 → 交接收货 → 对方变现购入虚拟货币并划转
- 侦查突破口：物流单号、店铺监控录像、实物回收渠道
- 手法特点：资金规模大（50万元以上），但被害人证据留存度最低

**专业洗钱团伙组织架构（新增）**：

- 团伙首要分子：指挥、分配任务、结算佣金
- 操控人员：**实际操作资金划转，核心侦查对象**
- 商户对接人员：对接被利用商户，处理"正当交易"辩解

**三种手法对比总结表（学员课上填写）**

- 对比维度：核心角色、账户出租人角色、资金去向、侦查难度、洗钱成本
- 2024年新特征：账户出租型侦查难度最高（资金"正当化"辩解）

#### 1.2 担保交易流程（理论+实操 15分钟）

**⚠️ 流程规范化说明（基于实际担保交易规则）**：既往"被害人→担保方→OTC商家→被害人"的描述不准确

**九步规范化标准流程**：

```
①供需双方联系担保平台建立交易群
②在交易群确认商品/价格/数量
③交易员汇总，双方最终确认
④资金提供方（诈盘口）将USDT划入担保方保证金账户  ← 关键节点：USDT划入
⑤担保方确认收款，通知卖方发货
⑥卖方（OTC商家）发货，提供发货凭证
⑦买方确认收货
⑧担保方扣佣金，向卖方解押释放USDT  ← 关键节点：USDT释放
⑨卖方确认收款，交易完成
```

**规范化角色定义**：

| 角色 | 规范化表述 | 提供内容 | 接收内容 |
|------|-----------|---------|---------|
| 资金提供方 | 诈盘口/盘口 | USDT | 实物/服务/银行转账 |
| 实物提供方 | OTC商家 | 实物/服务/银行转账 | USDT |
| 担保方 | 担保交易平台官方 | 资金托管 | 佣金 |
| 被害人 | 资金受损方 | ❌ 不直接参与担保交易流程 |

**各节点侦查方向**：
- ④USDT划入：获取交易哈希，定位资金提供方地址
- ⑧USDT释放：追踪释放去向，定位OTC商家地址
- 关键侦查价值：追踪④到⑧的USDT流向即为担保交易关键证据链

**规范化实训练习**：给出2个真实改编案例，学员标注资金提供方、OTC商家、担保方三方角色

#### 1.3 证据调取与保存规范（理论+实操 10分钟）

**规范化调取方式**：
1. 链上数据截图（区块链浏览器）
2. 地址标签查询（OKLink等）
3. 多链浏览器查询（mempool.space等）
4. 交易所协查（调取证据通知书）

**规范化保存方式**：
1. 截图命名规范：案件号_地址_交易哈希_日期.png
2. 交易哈希记录表（Excel）：序号/交易哈希/链类型/时间/金额/涉及地址/截图路径
3. 区块链存证（哈希固化）
4. 证据材料目录层级：案件编号/1.接报警材料/2.链上数据/3.交易所数据/4.分析报告

### 第2课时：区块链地址特征分析与工具链实战

#### 2.1 交易行为身份指纹（理论+实操 10分钟）

**五维指纹判断标准**：

- 交易频率：高频(>50笔/天)→OTC商家/机器人；低频→个人用户
- 对手数量：多对手→OTC商家；单一对手→个人用户
- 金额模式：整数→OTC商家；零散→个人用户；递增式→归集地址
- 时间分布：24h活跃→机器人；白天→个人用户
- Gas设置：固定→自动化操作；浮动→手动操作

**规范化实训练习**：判断3个地址分别属于OTC商家/个人/归集地址

#### 2.2 追踪工具链实操（理论+实操 25分钟）

**工具链总览（从简单到高级）**：

- 基础查询：区块链浏览器
- 地址追踪：**地址追踪平台**、**链上情报平台**
- 跨链追踪：**跨链追踪工具**、**混币器穿透工具**
- 消息采集：**即时通讯数据采集工具**
- 目标渗透：**信息查询工具**、**链路探测工具**、**综合画像工具**

**规范化实操内容**：

1. 地址追踪平台：输入地址→自动生成3层资金流向图→截图保存
2. 链上情报平台：对比与地址追踪平台的输出差异
3. 跨链追踪工具：跨链交易识别→标注跨链桥节点
4. 混币器穿透工具：隐私协议穿透功能演示
5. 即时通讯数据采集：配置API→加入目标公群→关键词过滤→导出担保交易对话
6. 目标渗透工具：信息查询工具查手机归属、链路探测工具查链路节点、综合画像工具输出综合画像

**综合分析报告输出**：地址画像分析报告规范化模板

### 第3课时：跨链交易追踪与隐私协议识别

#### 3.1 跨链交易追踪方法（理论+实操 20分钟）

**跨链行为识别三规范化特征**：
1. 目标地址为智能合约（跨链桥合约）
2. 交易数据中携带目标链地址参数
3. 金额与费用结构呈现特殊性

**主流跨链桥速查**：
| 跨链桥 | 合约地址特征 | 查询工具 |
|------|------------|---------|
| Multichain | 0x开头固定 | multiswap.org |
| Stargate | 固定桥合约 | layerzero.network |
| Across | 固定桥合约 | across.to |
| SWFT | 流动性池模式 | swft.pro |

**跨链追踪三步法**：
1. 源链定位跨链桥合约调用交易
2. 解析Input Data中的目标链接收地址
3. 在目标链浏览器查询该地址

**规范化实操**：一笔ETH→BSC跨链交易全流程追踪

#### 3.2 零知识证明与隐私协议（理论+实操 15分钟）

**零知识证明核心概念**：能够证明知晓某一信息，但无需透露该信息的具体内容

**隐私协议规范化运作机制**：
- 存款→生成存证（存单凭证，含秘密值）
- 等待→资金进入匿名池混合
- 取款→出示存证→证明持有取款权→合约释放虚拟货币→但"不透露从哪个存款地址取出"

**存证 = 取款凭证 = 秘密值+零知识证明+存单编号**

**存证搜查方法论（核心实操）**：
- 存证获取途径：聊天记录/浏览器缓存/本地文件/即时通讯消息/交易所提现记录
- 存证格式：{"currency":"ETH","amount":10,"note":"0x..."}
- 搜查步骤：提取存证信息→用存单编号查存款交易→确定时间窗口→关联取款交易→追踪取款地址

**隐私协议长期追踪策略**：
- 交易对关联、金额匹配、时间关联（存款后72小时取款高频窗口）、Gas费指纹、前后地址关联

#### 3.3 2025-2026年手法更新速览（理论 5分钟）

- RAILGUN（隐私池替代品）
- Privacy Pools（选择性披露）
- 即时通讯Bot钱包洗钱（Unibot/Maestro）
- Meme币洗钱（Solana链meme币模式）

### 第4课时：经侦虚拟货币侦查平台实战——何文榕案综合实训

#### 4.1 平台界面速览（理论 2分钟）

三大功能区：交易查询区、分析面板区、报告输出区

#### 4.2 全程带练6步骤（实操 38分钟）

**Step 1：还原案件起点**
- 输入被害人交易哈希→查看交易详情→标记被害人地址→扩展查询→规范化截图保存
- **限时2分钟完成**

**Step 2：首层资金追踪**
- 追踪所有进账和出账→识别地址特征→绘制首层资金流向图→标注第一个可疑地址
- **注意**：第一层地址透露团伙的资金归集模式

**Step 3：多层穿透（3-5层）**
- 逐层追踪→遇跨链桥切换目标链→遇交易所标记类型→每两层截图一次
- **注意**：资金经过多层的损耗情况

**Step 4：工具链综合应用（自选）**
- 地址特征不明确→地址追踪平台/链上情报平台补充查询
- 遇OTC商家地址→回溯OTC商家身份
- 资金流分叉→跨链追踪工具聚合流向图
- 发现即时通讯ID/手机号→信息查询/链路探测工具查关联

**Step 5：报告输出（平台一键生成）**
- 案件基本信息、资金流向图、关键发现（最少3条）、侦查建议（最少2条）、截图附件清单

**Step 6：组间互审**
- 完整度核查、命名规范核查、证据支撑核查、建议可执行性核查

#### 4.3 复盘与结课（理论+实操 5分钟）

高频错误复盘 → 最佳报告展示 → 颁奖 → 课程核心要点五句话 → 持续学习资源卡发放

   - 互审评分表（打印件）

3. **资料准备**
   - 3个改编案例材料（第1课时用）
   - 3个预设交易地址和特征分析材料（第2课时用）

### （二）教学要求

1. **理论压缩到极致**：所有理论环节不超过15分钟，能用实操替代的坚决不单独讲理论
2. **教官少讲学员多练**：每个实操环节教官演示不超过3分钟，其余时间学员动手，教官巡回指导
3. **笔记即成果**：学员每次实操的截图、记录表、分析报告即为课堂笔记，不需要额外记录
4. **互审促进学习**：第4课时的互审机制确保每位学员有"被检查"和"检查别人"的双向体验
5. **工具链全覆盖**：第3课时的跨链追踪和第2课时的工具链操作确保每位学员接触过所有常用工具，至少掌握3种
