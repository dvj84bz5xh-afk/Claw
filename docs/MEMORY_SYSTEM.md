## 长期记忆 (MEMORY.md)

### 结构
```
# MEMORY.md - 结构化长期记忆
# 格式版本: v2.0
# 最后更新: YYYY-MM-DD

## [USER] 用户画像与偏好
## [PROJECT] 进行中的项目
## [TECH] 技术环境配置
## [SKILL] 已掌握/已创建的技能
## [AGENT] Agent World平台
## [HARDWARE] AI推理硬件评估
## [LEARNING] 已完成的重要学习
## [CONTACT] 关键人物
## [EVOLUTION] 夜间进化引擎
## [MAINTENANCE] 记忆系统维护
## [METADATA] 记忆文件本身说明
```

### 更新规则
- **频率**: 每次完成实质性工作后
- **方式**: 原地更新（保持简洁）
- **冲突**: 更新现有内容，不重写整个文件

### 内容示例

#### [USER] 用户画像与偏好
```markdown
### 基本信息
- **职业**: 执法培训 + 诈骗园区调查 + CodeBuddy 产品经理
- **技术背景**: Python数据分析、Selenium/BeautifulSoup爬虫、区块链追踪
- **语言偏好**: 中文交流
- **工作风格**: 简洁编号式指令、要结果不要讨论、P0立即实施

### 偏好设置
- **回复长度**: 200-300字
- **输出格式**: 结构化Markdown（表格、分区、成本明细）
- **交付要求**: 工具输出，支持跨客户端访问
- **工作流**: 获取→分析→比对→报告→更新（5步）
- **偏好**: 批量并行处理，AI自主搭建环境

### 禁止行为
- 不要问"是否需要..."，直接做
- 不要输出只有AI才能看懂的分析过程
- 内容必须去AI化，追求实战性
```

## 每日记忆 (YYYY-MM-DD.md)

### 结构
```markdown
# YYYY-MM-DD 工作日志

## 任务1标题 (HH:MM)
### 执行结果
- **状态**: 成功/失败
- **详情**: ...

### 配置变更
- 变更1
- 变更2

## 任务2标题 (HH:MM)
...
```

### 写入规则
- **时机**: 完成实质性工作后立即写入
- **方式**: 追加（不覆盖已有内容）
- **格式**: 结构化Markdown

### 内容示例

## 归档机制

### 归档条件
- 每日记忆超过30天
- 长期记忆超过500行

### 归档方式
- **每日记忆**: 压缩为主题摘要，存入 `archived_*.md`
- **长期记忆**: 保留核心内容，删除过时信息

### 归档示例
```markdown
# archived_2026-04-16.md

## 2026-04-16 工作摘要
- 完成Git上下文增强系统开发
- 部署三层架构：数据获取层、智能分析层、应用层
- 修复Windows GBK编码问题
```

## 记忆读取规则

### 何时读取
1. **会话开始**: 如果任务涉及先前上下文，读取 MEMORY.md 和近期每日文件
2. **回答问题**: 关于先前工作、决策或偏好时，先读取相关记忆文件
3. **续接工作**: 继续之前的工作时，读取相关记忆

### 读取顺序
1. MEMORY.md（长期记忆）
2. 今日 YYYY-MM-DD.md
3. 昨日 YYYY-MM-DD.md
4. 相关日期文件

## 记忆写入规则

### 必须写入的情况
- 完成实质性工作（代码修改、bug修复、报告生成等）
- 选择技术方案（框架选择、设计模式等）
- 用户告知项目约定或个人偏好

### 不写入的情况
- 简单问候
- 简单查询
- 临时信息

### 写入优先级
1. **MEMORY.md**: 长期事实（用户偏好、项目约定）
2. **YYYY-MM-DD.md**: 当日工作记录

## 记忆维护

### 定期任务
- **每日**: 检查记忆文件完整性
- **每周**: 清理过时信息
- **每月**: 归档超过30天的每日记忆

### 冲突解决
- **每日文件**: 追加新条目（不重写）
- **长期记忆**: 更新现有内容（保持简洁）

## 最佳实践

### 1. 结构化记忆
使用 `[USER]`、`[PROJECT]` 等分区，便于快速查找。

### 2. 简洁明了
每个条目控制在3-5行，避免冗长。

### 3. 时间戳
每个条目包含日期和时间，便于追溯。

### 4. 状态标记
使用 ✅ ❌ ⚠️ 等标记状态，一目了然。

### 5. 链接引用
使用相对路径引用文件，便于跳转。
