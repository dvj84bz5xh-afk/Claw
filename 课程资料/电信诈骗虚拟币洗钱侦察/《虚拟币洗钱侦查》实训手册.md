## 实训一：区块链浏览器基础操作

### 实训目标
- 掌握主流区块链浏览器的使用方法
- 能够查询交易详情和地址信息
- 学会识别交易类型和状态

### 实训环境
- 计算机（可联网）
- 浏览器（Chrome/Edge/Firefox）
- 区块链浏览器访问权限

### 常用区块链浏览器

| 区块链 | 浏览器网址 | 适用范围 |
|--------|-----------|---------|
| 以太坊 | etherscan.io | ETH及ERC20代币 |
| 波场 | tronscan.org | TRX及TRC20代币（USDT-TRC20） |
| 币安智能链 | bscscan.com | BNB及BEP20代币 |
| 比特币 | blockchain.com/explorer | BTC |
| 欧科云链 | oklink.com | 多链支持（推荐） |

### 实训步骤

#### 步骤1：访问区块链浏览器
1. 打开浏览器，访问 https://www.oklink.com
2. 观察首页布局：搜索框、行情数据、最新区块等
3. 熟悉各功能模块位置

#### 步骤2：查询交易信息
1. 在搜索框输入示例交易哈希：
   ```
   0x7a8c2f9d3e1b5a6c4d8e7f9a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9a0b
   ```
2. 查看交易详情页面：
   - 交易状态（Success/Pending/Failed）
   - 区块高度（Block Number）
   - 时间戳（Timestamp）
   - 发送方地址（From）
   - 接收方地址（To）
   - 交易金额（Value）
   - Gas费用（Gas Fee）

#### 步骤3：查询地址信息
1. 复制发送方地址，在搜索框查询
2. 查看地址详情：
   - 地址余额
   - 交易总数
   - 交易历史列表
3. 分析该地址的交易模式

#### 步骤4：识别代币转账
1. 找到Token Transfer（代币转账）记录
2. 识别代币类型（USDT、USDC等）
3. 查看代币合约地址

### 实训要求
- 完成至少3个不同交易的查询
- 记录查询结果到实训报告
- 截图保存关键页面

### 注意事项
1. 所有操作均为查询，不要进行任何转账操作
2. 截图时注意不要暴露敏感信息
3. 遇到加载缓慢时请耐心等待

## 实训二：链上资金追踪实操

### 实训目标
- 掌握链上资金追踪的基本方法
- 学会绘制资金流向图
- 能够识别关键节点（交易所、混币器等）

### 实训案例（模拟数据）

**案情简介**：
受害人张三被"杀猪盘"诈骗，向嫌疑人提供的地址转账USDT。经调查，受害人转账记录如下：

- 转账时间：2024年3月15日 14:30
- 转账金额：50,000 USDT
- 受害人地址：0xVictimA1B2C3D4E5F6...
- 嫌疑人地址：0xSuspectX1Y2Z3W4...

### 实训步骤

#### 步骤1：获取初始信息
1. 记录受害人转账的交易哈希
2. 在区块链浏览器查询该交易
3. 确认交易状态为Success
4. 记录交易时间（转换为北京时间）

#### 步骤2：第一层追踪
1. 查看嫌疑人地址（0xSuspectX1Y2...）的余额和交易历史
2. 记录该地址在收到受害人资金后的所有转出交易
3. 识别转出地址的类型（个人钱包/交易所/混币器）

#### 步骤3：多层追踪
1. 对第一层转出地址继续追踪
2. 绘制资金流向图（至少3层）
3. 标注每个节点的类型和金额

#### 步骤4：识别关键节点
使用地址标签库识别以下类型地址：
- 交易所地址（Binance、OKX、Huobi等）
- 混币器地址（Tornado Cash等）
- DeFi协议地址（Uniswap等）
- 跨链桥地址

### 资金流向图绘制规范

```
示例格式：

受害人地址
    │
    │ 50,000 USDT
    ▼
嫌疑人地址（第一层）
    │
    ├─ 20,000 USDT ──► 地址A（Binance交易所）
    │
    ├─ 15,000 USDT ──► 地址B（Tornado Cash混币器）
    │                      │
    │                      ▼
    │                   地址C（混币后）
    │                      │
    │                      ▼
    │                   地址D（OKX交易所）
    │
    └─ 15,000 USDT ──► 地址E（个人钱包）
```

### 实训要求
1. 完成至少3层资金流向追踪
2. 绘制完整的资金流向图
3. 撰写追踪分析报告，包括：
   - 资金流向概述
   - 关键节点识别结果
   - 可疑交易特征
   - 下一步侦查建议

## 实训三：证据固定实操

### 实训目标
- 掌握电子证据固定的规范流程
- 学会使用区块链存证工具
- 能够制作符合法律要求的证据材料

### 法律依据
- 《公安机关办理刑事案件电子数据取证规则》
- 《最高人民法院关于互联网法院审理案件若干问题的规定》
- 《电子数据司法鉴定通用实施规范》

### 实训步骤

#### 步骤1：截图取证规范
1. 打开区块链浏览器，显示完整交易详情页面
2. 按以下要求进行截图：
   - 包含完整网址（地址栏可见）
   - 包含系统时间（右下角时间可见）
   - 包含交易关键信息（TxHash、地址、金额、时间）
3. 使用截图工具（Snipaste/ShareX）进行标注：
   - 用红色框标注关键信息
   - 添加文字说明

#### 步骤2：录屏取证（可选）
对于复杂操作流程，使用录屏软件记录：
1. 打开OBS Studio或类似录屏软件
2. 设置录制区域为浏览器窗口
3. 开始录制后，完整展示查询操作过程
4. 录制完成后，计算文件哈希值（SHA256）

#### 步骤3：区块链存证
使用第三方区块链存证平台进行证据固化：

**常用存证平台**：
- 蚂蚁区块链（蚂蚁链）
- 至信链
- 易保全
- 保全网

**存证步骤**：
1. 访问存证平台官网
2. 注册并实名认证
3. 上传截图/文档
4. 获取存证证书和哈希值
5. 保存存证编号和查询链接

#### 步骤4：制作《电子证据检查笔录》

```
电子证据检查笔录

检查地点：________________________
检查人员：________________________

二、检查过程
1. 打开区块链浏览器：https://www.oklink.com
2. 在搜索框输入交易哈希进行查询
3. 记录查询结果如下：
   - 交易状态：____________________
   - 区块高度：____________________
   - 交易时间：____________________
   - 发送地址：____________________
   - 接收地址：____________________
```

### 实训要求
1. 完成至少3笔交易的截图取证
2. 至少使用1个区块链存证平台进行存证
3. 制作完整的《电子证据检查笔录》

## 实训四：模拟案件综合分析

### 实训目标
- 综合运用所学知识分析模拟案件
- 完成从线索发现到侦查建议的完整流程
- 锻炼团队协作和汇报能力

### 分组安排
- 每组4-5人
- 设组长1人，负责统筹协调
- 设记录员1人，负责记录分析过程
- 设汇报员1人，负责最终汇报

### 模拟案件材料

**案件名称**："3·15"特大电信网络诈骗案

**基本案情**：
2024年3月15日，受害人李某（女，35岁，某公司财务）报警称被诈骗128万元。经初查，李某在某社交平台认识嫌疑人，对方以"内部投资渠道"为名，诱导李某向指定账户转账。

**涉案资金流**：
1. 李某银行账户 → 嫌疑人提供的银行账户（一级卡）：128万元
2. 一级卡 → 多个二级卡（分散转出）
3. 二级卡 → 地下钱庄（兑换为USDT）
4. 地下钱庄 → 多个虚拟币地址

**已获取的虚拟币地址**：
- 地址1：0xA1B2C3D4E5F6...（收到40万USDT）
- 地址2：0xG7H8I9J0K1L2...（收到35万USDT）
- 地址3：0xM3N4O5P6Q7R8...（收到53万USDT）

### 实训任务

#### 任务1：资金流向分析（20分钟）
1. 各组使用区块链浏览器查询三个地址的交易记录
2. 追踪资金后续流向（至少追踪3层）
3. 绘制资金流向图
4. 识别关键节点（交易所、混币器等）

#### 任务2：情报分析（15分钟）
1. 分析各地址的交易时间规律
2. 识别关联地址（共同转入/转出的地址）
3. 判断是否存在组织化作案特征
4. 评估资金追回可能性

#### 任务3：侦查方案设计（15分钟）
1. 制定下一步侦查方向
2. 列出需要调取的证据清单
3. 设计抓捕和追赃方案
4. 评估跨境协作需求

#### 任务4：成果汇报（每组5分钟）
1. 资金流向图展示
2. 关键发现汇报
3. 侦查方案说明
4. 回答提问

### 评价标准

| 评价维度 | 分值 | 评分标准 |
|---------|------|---------|
| 分析完整性 | 30分 | 资金流向图完整，层级清晰，关键节点识别准确 |
| 方法规范性 | 25分 | 查询方法正确，证据固定规范，工具使用熟练 |
| 方案可行性 | 25分 | 侦查方案切实可行，步骤清晰，考虑周全 |
| 汇报表现 | 20分 | 表达清晰，逻辑严密，时间控制得当，应答准确 |

## 实训六：资金流+通讯流双轨分析

### 实训目标
- 掌握资金流与通讯流关联分析方法
- 能够进行时间、金额、地址三维关联
- 学会绘制双轨追踪图

### 实训案例（模拟数据）

**案情简介**：
受害人王某报警称被"杀猪盘"诈骗。通过调查获取以下信息：

**通讯流信息**：
- 嫌疑人微信：LoveInvest2024
- 诱导话术："今晚8点有大行情，充值5万USDT，收益30%"
- 提供的收款地址：0xFraudAddress123...
- 聊天时间线：
  - 3月15日 20:00：发送收款地址
  - 3月15日 20:15：催促转账
  - 3月15日 20:30：确认收款

**资金流信息**：
- 受害人地址：0xVictimABC...
- 嫌疑人地址：0xFraudAddress123...
- 交易哈希：0xTxHashXYZ...
- 交易时间：2024-3-15 20:32
- 交易金额：50,000 USDT
- 后续流向：
  - 20:35 → 0xLayer2A...（20,000 USDT）
  - 20:38 → 0xLayer2B...（15,000 USDT）
  - 20:40 → 0xLayer2C...（15,000 USDT）

### 实训任务

#### 任务1：时间关联分析
1. 对比通讯时间和交易时间
2. 绘制时间对照表
3. 验证时间一致性

#### 任务2：金额关联分析
1. 对比话术中的金额和实际转账金额
2. 分析金额是否匹配
3. 查找其他关联交易

#### 任务3：地址关联分析
1. 确认通讯中提供的地址与实际收款地址
2. 分析地址后续流向
3. 识别关键节点（交易所、混币器）

#### 任务4：绘制双轨追踪图
使用以下模板绘制完整的双轨追踪图：

```
通讯流                    资金流
  │                        │
  ▼                        ▼
微信联系 ─────时间关联────→ 未转账
  │                        │
  ▼                        ▼
发送地址 ─────地址关联────→ 准备转账
  │                        │
  ▼                        ▼
催促转账 ─────时间关联────→ 正在转账
  │                        │
  ▼                        ▼
确认收款 ─────金额关联────→ 50,000 USDT到账
                              │
                              ▼
                         分层转出
                              │
                              ├─ 20,000 → 交易所A
                              ├─ 15,000 → 混币器
                              └─ 15,000 → 个人钱包
```

### 实训要求
1. 完成三维关联分析表
2. 绘制双轨追踪图
3. 撰写分析报告，包括：
   - 关联分析结论
   - 证据链完整性评估
   - 下一步侦查建议

## 实训七：情报研判与串并案分析

### 实训目标
- 掌握案件情报收集方法
- 学会地址关联分析
- 能够发现串并案线索

### 实训案例库（模拟数据）

**案件A（2024-01-15）**：
- 涉案地址：0xA1、0xA2
- 诈骗类型：杀猪盘
- 受害人：张某
- 损失：80,000 USDT

**案件B（2024-02-20）**：
- 涉案地址：0xB1、0xB2
- 诈骗类型：杀猪盘
- 受害人：李某
- 损失：100,000 USDT

**案件C（2024-03-10）**：
- 涉案地址：0xC1、0xC2
- 诈骗类型：虚假投资
- 受害人：王某
- 损失：60,000 USDT

**链上发现**：
- 0xA2 向 0xB1 转账 30,000 USDT
- 0xB2 向 0xC1 转账 20,000 USDT
- 0xA1、0xB1、0xC1 均向同一交易所地址充值

### 实训任务

#### 任务1：地址关联图谱绘制
1. 绘制三个案件的地址关系图
2. 标注资金流向
3. 识别共同地址

#### 任务2：串并案分析
1. 分析地址关联关系
2. 判断是否符合串并案条件
3. 制作串并案报告

#### 任务3：团伙画像
1. 分析作案时间规律
2. 分析资金归集模式
3. 推断组织架构

### 串并案条件检查清单

| 条件 | 案件A vs B | 案件B vs C | 结论 |
|------|-----------|-----------|------|
