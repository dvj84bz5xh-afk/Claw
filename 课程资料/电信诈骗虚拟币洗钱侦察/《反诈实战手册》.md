## 第一部分：接警快速处置

### 一、接警询问要点（黄金5分钟）

#### 必问问题清单

| 序号 | 问题 | 目的 | 获取信息 |
|------|------|------|----------|
| 1 | 您是怎么转账的？银行卡还是虚拟币？ | 确定资金通道 | 法币/虚拟币 |
| 2 | 如果是虚拟币，是什么币种？USDT还是其他的？ | 确定币种 | USDT/BTC/ETH等 |
| 3 | 您转账的地址是什么？有没有截图？ | 获取涉案地址 | 虚拟币地址 |
| 4 | 有没有交易哈希（TxHash）？ | 唯一标识交易 | TxHash |
| 5 | 什么时候转的？转了多少钱？ | 确定金额和时间 | 金额、时间 |
| 6 | 对方的联系方式是什么？ | 获取通讯线索 | 电话、微信、QQ等 |
| 7 | 有没有安装对方提供的APP？ | 判断诈骗类型 | 虚假APP |
| 8 | 对方是怎么联系上您的？ | 确定引流渠道 | 社交平台、短信等 |

#### 快速判断流程图

```
接到报警
    │
    ▼
询问转账方式
    │
    ├─ 银行卡转账 ──► 传统电诈流程（启动紧急止付）
    │
    └─ 虚拟币转账 ──► 获取地址/TxHash
              │
              ▼
        区块链浏览器查询
              │
              ├─ 资金未转出 ──► 持续监控，尝试联系交易所
              │
              └─ 资金已转出 ──► 追踪去向，判断是否进入交易所
                        │
                        ├─ 进入交易所 ──► 紧急发协查函冻结
                        │
                        └─ 进入混币器 ──► 标记监控，分析破解
```

### 二、紧急止付操作指引

#### 虚拟币案件止付特点

**与传统案件的区别**：
- ❌ 无法通过银行系统止付
- ❌ 无法通过支付平台冻结
- ✅ 只能通过交易所冻结
- ✅ 可通过监控地址掌握动态

#### 止付操作步骤

**第一步：确认资金状态（10分钟内）**
1. 获取涉案地址和交易哈希
2. 使用OKLink/Etherscan查询交易状态
3. 确认资金当前所在地址
4. 判断资金是否已进入交易所

**第二步：如资金进入交易所（30分钟内）**
1. 立即准备《紧急冻结函》
2. 联系交易所合规部门（电话+邮件）
3. 发送警官证、立案决定书、冻结函
4. 电话确认收到并处理
5. 跟进冻结结果

**第三步：如资金未进入交易所**
1. 将涉案地址录入监控列表
2. 设置资金变动提醒
3. 一旦资金进入交易所立即启动冻结
4. 同步开展链上追踪

## 第二部分：常见诈骗话术与破解

### 一、杀猪盘话术破解

#### 典型话术

**建立信任阶段**：
- "我在国外做IT/金融/做生意"
- "平时工作很忙，想找个真心的人"
- "我的妻子/丈夫去世了，孩子跟着父母"

**诱导投资阶段**：
- "我这里有内部投资渠道"
- "我自己也在投，收益很稳定"
- "今晚有个大行情，错过了就没有了"
- "你先试试小额，赚了再加大"

**收割阶段**：
- "系统维护，暂时无法提现"
- "需要缴纳保证金/解冻费"
- "你的账户异常，需要验证"

#### 破解要点

| 话术类型 | 破解方法 | 询问要点 |
|----------|----------|----------|
| 高富帅人设 | 核实身份信息 | 让对方视频通话，看是否会拒绝 |
| 内部渠道 | 强调正规性 | 问平台名称，查询是否有资质 |
| 限时机会 | 制造紧迫感 | 让受害人拖延，看对方反应 |
| 无法提现 | 识别套路 | 这就是典型的诈骗信号 |

### 二、刷单返利话术破解

#### 典型话术

**引流阶段**：
- "招兼职，日赚300-500"
- "点点赞就能赚钱"
- "不收任何费用，免费培训"

**小额返利阶段**：
- "先完成3单小额任务"
- "每单返利20%，做完就返"
- "这是个连单，需要完成所有任务"

**大额收割阶段**：
- "系统卡单，需要重新支付"
- "操作失误，需要补单"
- "最后一个大额任务，做完一起返"

#### 破解要点

**关键识别点**：
1. 任何要求先垫付资金的"兼职"都是诈骗
2. 正常刷单违法，不存在"正规刷单"
3. 连单任务、系统卡单都是话术
4. 返利比例超过10%就要警惕

### 三、冒充公检法话术破解

#### 典型话术

**恐吓阶段**：
- "我是XX公安局的，你涉嫌洗钱"
- "你的身份证被盗用了"
- "不配合调查就要逮捕你"
- "这是国家机密，不能告诉任何人"

**隔离阶段**：
- "不能挂电话，通话会被录音"
- "去酒店开房，不要联系家人"
- "手机设置呼叫转移"

**转账阶段**：
- "资金要转到安全账户核查"
- "虚拟币也要清查"
- "清查完会退给你"

#### 破解要点

**核心提醒**：
- ⚠️ 公安机关不会电话办案
- ⚠️ 不存在"安全账户"
- ⚠️ 不会要求转账到虚拟币地址
- ⚠️ 真正的警察会上门出示证件

**应对措施**：
1. 立即挂断电话
2. 拨打110或到派出所核实
3. 联系家人确认安全
4. 保存通话录音作为证据

## 第三部分：资金流+通讯流双重追踪

### 一、双轨追踪理念

#### 为什么需要双重追踪？

```
单一资金流追踪的问题：
- 只能知道钱去哪了
- 无法知道谁操作的
- 无法关联其他案件

单一通讯流追踪的问题：
- 只能知道谁联系的
- 无法知道资金规模
- 无法确定犯罪金额

双轨追踪的优势：
✓ 人+钱双向锁定
✓ 相互印证证据
✓ 发现关联案件
✓ 确定团伙结构
```

#### 双轨追踪流程

```
案件线索
    │
    ├──► 资金流追踪 ──► 发现涉案地址 ──► 链上分析
    │                                          │
    │                                          ▼
    │                                   资金流向图
    │                                          │
    │                                          ▼
    │                                   交易所KYC ──► 嫌疑人身份
    │
    └──► 通讯流追踪 ──► 发现社交账号 ──► 账号分析
                                               │
                                               ▼
                                        登录日志、IP地址
                                               │
                                               ▼
                                        关联手机、实名信息
                                               │
                                               ▼
                                        嫌疑人身份
                                               │
                                               ▼
                                        双向印证 ──► 锁定嫌疑人
```

### 二、通讯流追踪方法

#### 社交账号分析

**获取账号信息**：
- 微信：微信号、绑定手机号、注册时间
- QQ：QQ号、绑定信息、登录记录
- Telegram：用户名、注册信息
- 其他APP：账号ID、绑定信息

**账号关联分析**：
- 同一手机号注册的多个账号
- 同一设备登录的多个账号
- 同一IP地址登录的账号群

#### 通讯内容分析

**关键信息提取**：
- 诱导转账的话术和理由
- 提供的收款地址/账户
- 投资平台名称和链接
- 团伙组织架构信息

**时间线梳理**：
- 首次接触时间
- 建立信任阶段时长
- 诱导转账时间点
- 资金转移时间

### 三、资金流+通讯流关联

#### 关联方法

**时间关联**：
- 通讯中提到的转账时间 vs 实际交易时间
- 发现时间差是否一致

**金额关联**：
- 通讯中提到的金额 vs 实际转账金额
- 确认是否为同一笔交易

**地址关联**：
- 通讯中提供的地址 vs 实际收款地址
- 确认嫌疑人控制范围

#### 关联分析示例

```
案例：杀猪盘案件

通讯流发现：
- 嫌疑人微信号：xxx
- 诱导受害人于2024.3.15 14:30转账
- 提供的收款地址：0xABC...

资金流发现：
- 2024.3.15 14:32有50,000 USDT转入0xABC
- 该地址为某交易所充值地址
- 后续资金转入嫌疑人控制的其他地址

关联结论：
✓ 时间匹配（14:30通讯 vs 14:32交易）
✓ 地址匹配（通讯提供 vs 实际收款）
✓ 金额匹配（约定的5万元 vs 实际转账）
✓ 形成完整证据链
```

## 第五部分：情报研判与团队协作

### 一、案件情报研判

#### 情报收集清单

| 情报类型 | 来源 | 价值 |
|----------|------|------|
| 涉案地址 | 受害人、嫌疑人 | 核心追踪线索 |
| 社交账号 | 受害人手机 | 人员身份线索 |
| 银行流水 | 银行调取 | 法币通道线索 |
| 虚拟币交易 | 交易所调取 | 实名身份线索 |
| 通讯记录 | 运营商调取 | 关联人员线索 |
| IP地址 | 平台日志 | 地理位置线索 |

#### 情报分析方法

**地址关联分析**：
```
目的：发现多个案件是否关联

方法：
1. 收集历史案件涉案地址库
2. 比对新案件地址
3. 发现地址重合 → 串并案件
4. 发现共同转入地址 → 发现团伙

示例：
案件A地址：0xA1, 0xA2
案件B地址：0xB1, 0xB2
发现0xA2和0xB1有资金往来
→ 案件A和B可能关联
```

**时间序列分析**：
```
目的：发现作案规律

方法：
1. 整理多个案件时间线
2. 分析作案时间分布
3. 发现集中作案时段
4. 推断团伙作息规律

示例：
案件A：2024.3.15 14:30
案件B：2024.3.15 15:10
案件C：2024.3.15 16:00
→ 团伙可能在同一时区，下午活跃
```

### 三、串并案研判

#### 串并案条件

**必须同时满足以下条件**：
1. 同一涉案地址出现在多个案件
2. 同一社交账号出现在多个案件
3. 同一收款账户出现在多个案件
4. 作案手法高度相似
5. 作案时间存在关联

#### 串并案流程

```
发现疑似关联
    │
    ▼

    │
    ▼
层报上级审批
    │
    ▼
成立专案组
    │
    ▼
统一指挥、并案侦查
```

## 第六部分：刑事法律适用

### 一、罪名认定

#### 常见罪名

| 罪名 | 适用情形 | 量刑标准 |
|------|----------|----------|
| 诈骗罪 | 以非法占有为目的，骗取财物 | 数额较大：3年以下；巨大：3-10年；特别巨大：10年以上/无期 |
| 掩饰隐瞒犯罪所得罪 | 明知是犯罪所得而协助转移 | 3年以下；情节严重：3-7年 |
| 帮助信息网络犯罪活动罪 | 为诈骗提供技术支持、支付结算 | 3年以下 |
| 非法利用信息网络罪 | 发布诈骗信息、建立诈骗网站 | 3年以下 |
| 洗钱罪 | 掩饰隐瞒毒品、黑社会等犯罪所得 | 5年以下；情节严重：5-10年 |

#### 罪名选择要点

**主犯**：
- 组织、领导诈骗团伙 → 诈骗罪主犯

**从犯**：
- 提供技术支持 → 帮助信息网络犯罪活动罪
- 提供支付结算 → 掩饰隐瞒犯罪所得罪/帮助信息网络犯罪活动罪
- 引流拉新 → 诈骗罪从犯

**虚拟币洗钱人员**：
- 明知是诈骗所得而协助转移 → 掩饰隐瞒犯罪所得罪
- 与诈骗团伙通谋 → 诈骗罪共犯

### 二、证据标准

#### 定罪证据清单

**主体证据**：
- 嫌疑人身份证明
- 社交账号实名信息
- 交易所KYC资料
- 通讯工具注册信息

**客观行为证据**：
- 聊天记录（诈骗话术）
- 虚假平台资料
- 转账记录（银行+虚拟币）
- 链上资金追踪记录
- 资金流向图

**主观故意证据**：
- 话术培训资料
- 分工协作记录
- 获利分配记录
- 嫌疑人供述

**金额证据**：
- 受害人陈述
- 转账凭证
- 链上交易记录
- 虚拟币价值证明

#### 电子证据固定规范

**必须满足的要求**：
1. 取证主体合法（两名以上侦查员）
2. 取证程序合法（搜查证、扣押证）
3. 取证手段合法（不得篡改、删除）
4. 取证记录完整（笔录、见证人）
5. 证据保存安全（原始介质封存）

**虚拟币证据固定要点**：
- 区块链浏览器截图（含网址、时间）
- 交易详情完整记录
- 资金流向图
- 区块链存证（如有）
- 价格认定（案发时虚拟币价值）

### 三、管辖与协作

#### 管辖确定

**地域管辖**：
- 犯罪行为地：诈骗行为实施地、虚拟币转移地
- 犯罪结果地：受害人所在地、资金损失地
- 优先管辖：主要犯罪地或主要受害人所在地

**级别管辖**：
- 一般案件：县级公安机关
- 重大案件：市级公安机关
- 特别重大案件：省级公安机关

#### 内部协作

**协作请求内容**：
1. 案件基本情况和管辖依据
2. 需要协作的具体事项
3. 相关法律文书
4. 联系人和联系方式

**常见协作事项**：
- 调取通讯记录
- 查询银行流水
- 冻结银行账户
- 抓捕嫌疑人
- 查封服务器
