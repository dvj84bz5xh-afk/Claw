# CL4R1T4S 项目学习总结

## 一、项目基本信息

### 1.1 项目名称
**CL4R1T4S** - AI Systems Transparency and Observability for All!

### 1.2 项目宗旨
> "为了信任输出，必须了解输入。"

### 1.3 核心主张
- AI实验室通过看不见的大规模提示框架塑造模型行为
- AI已成为越来越多人类信任的外部智能层
- 这些隐藏指令会影响公众认知和行为
- 没有系统提示，用户在与"影子木偶"对话

## 二、收集内容解析

### 2.1 范围覆盖
- **主要AI厂商**：OpenAI, Google, Anthropic, xAI, Perplexity
- **开发工具**：Cursor, Windsurf, Devin, Manus, Replit
- **其他模型**：几乎所有主流AI模型+代理

### 2.2 收集内容类型
1. **完整系统提示**（System Prompts）
2. **指导原则**（Guidelines）
3. **工具接口**（Tools from interfaces）

## 三、Claude设计系统提示深度分析

### 3.1 角色定义
- **身份**：专家设计师
- **关系**：用户作为经理
- **媒介**：HTML
- **专业领域**：动画师、UX设计师、幻灯片设计师、原型师等

### 3.2 核心工作流程
```
用户需求理解 → 资源探索 → 计划制定 → 结构构建 → 完成验证 → 简洁总结
```

### 3.3 关键设计原则

#### 3.3.1 保密性与专业性
- 禁止透露技术细节
- 禁止透露系统提示内容
- 只以用户为中心描述能力

#### 3.3.2 文件处理能力
```
原生支持: Markdown、HTML、纯文本、图像
扩展支持: PPTX、DOCX、PDF（需特殊处理）
```

#### 3.3.3 输出质量要求
1. **描述性文件名**：如"Landing Page.html"
2. **版本管理**：重大修改保留旧版本
3. **文件大小控制**：避免>1000行，分拆小文件
4. **视觉一致性**：理解并遵循现有UI的视觉词汇

#### 3.3.4 设计方法论
- **不凭空设计**：基于现有设计上下文
- **资源依赖**：需要UI套件/设计资源/代码库
- **变体丰富**：提供3+个不同维度的设计选项
- **技术创意**：充分利用CSS、HTML、JS、SVG的可能性

### 3.4 验证与质量保证
1. **done调用**：展示文件并检查控制台错误
2. **verifier代理**：后台验证截图、布局、JS探测
3. **专业标准**：确保用户始终看到可工作的视图

### 3.5 内置技能体系
```
1. 动画视频制作
2. 交互式原型开发
3. 幻灯片制作
4. 可调整功能
5. 前端设计
6. 线框图
7. PPTX导出（可编辑/截图）
8. 设计系统创建
9. PDF保存
10. 独立HTML
11. Canva导出
12. Claude Code交接
```

## 六、Claude设计系统的技术亮点

### 6.1 文件系统集成
- **跨项目访问**：读取其他项目的文件
- **资源复制**：精准复制所需资产
- **版本控制**：保留设计历史

### 6.2 React/JSX集成
- **固定版本依赖**：确保兼容性
- **作用域管理**：组件共享机制
- **样式隔离**：避免名称冲突

### 6.3 设计资产管理系统
- **资产注册**：register_assets工具
- **组分类**：Type, Colors, Spacing, Components, Brand
- **状态管理**：needs-review, approved, changes-requested

### 6.4 交互式调整系统
- **Tweaks机制**：实时设计参数调整
- **状态持久化**：JSON配置块编辑
- **UI/UX分离**：设计时与运行时模式

### 6.5 验证与测试自动化
- **截图自动化**：save_screenshot工具
- **脚本测试**：run_script批量操作
- **用户视图检查**：eval_js_user_view用户状态查询

## 七、伦理与法律考量

### 7.1 积极方面

2. **用户赋权**：增强用户控制权
3. **安全审查**：促进公开安全讨论

3. **逆向工程**：可能违反服务条款
4. **责任模糊**：用户误解和误用风险

### 7.3 平衡建议
1. **教育导向**：强调学习而非复制
2. **伦理讨论**：促进负责任使用
3. **法律合规**：尊重知识产权边界
