## 1.1 三种常见回U手法详解

### 1.1.1 为什么诈骗团伙需要"回U"？

在开始学习具体手法前，必须理解一个核心问题：**为什么诈骗团伙不直接使用受害人的银行卡资金？**

**答案**：
1. **银行卡是"实名制"的** —— 资金一旦进入银行卡，公安机关可以通过银行流水快速追溯到实名信息
2. **电诈案件有"冻结机制"** —— 受害人报警后，公安机关可以快速对涉案银行卡进行紧急止付和冻结
3. **洗钱需要"匿名化"** —— 只有把资金转换成USDT（最主流的稳定币），才能通过区块链实现匿名转移

> 💡 **核心公式**：诈骗资金 + 回U手法 + 公群担保 = 难以追踪的跨境洗钱

**⚠️ 重要修正：卡主的真实角色**

根据实战案例分析，**卡接回U的真实运作模式**与早期理解有本质区别：

| 角色 | 表面理解 | 实际情况 | 侦察重点 |
|------|---------|---------|---------|
| **卡主** | 主动买U转给团伙 | 仅提供银行卡，**实际操作由洗钱车队完成** | 卡主是证人，不是主犯 |
| **洗钱车队** | ❌ 未被识别 | **核心角色**：操作员实际操作所有转账 | ⭐ 重点侦察对象 |
| **境内商家** | 正常经营 | 钱款最终转入，被要求提供"供货材料"证明 | 需深入追查资金来源 |

**完整运作流程（修正版）**：

1. **受害人**被骗后，将资金转入**卡主银行卡**
2. **卡主**仅提供银行卡（收取少量"卡费"），**不参与实际操作**
3. **洗钱车队操作员**获取卡主银行卡信息后，实际操作转账
4. 操作员通过OTC渠道购买USDT，或直接将钱款转到其他账户
5. **钱款最终去向**：转入**境内合法商家**的个人银行卡账号
6. 商家收到款项后，**解释为"境外客户使用地下钱庄向其转账"**，并能提供供货材料证明
7. **少部分个人**解释为"场外换U"，每U赚几分钱差价

**洗钱车队运作模式详解**：

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                            洗钱车队内部结构                                       │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   「车队老板」── 负责整体指挥、分配任务、结算佣金                                   │
│         │                                                                        │
│         ├── 「操作员A」── 操作卡主银行卡，转账到指定账户                           │
│         ├── 「操作员B」── 管理OTC渠道，买卖USDT                                   │
│         ├── 「车行」── 提供对公账户或商户二维码收款                               │
│         └── 「客服」── 对接卡主、商家，处理解释话术                               │
│                                                                                 │
│   佣金结构：                                                                     │
│   - 卡主：每张卡每月500-2000元"租卡费"                                           │
│   - 操作员：每笔转账0.5%-1%提成                                                  │
│   - 车队老板：每笔交易1%-3%抽成                                                  │
│   - 商家：提供"正当交易"证明，配合解释资金来源                                     │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**钱款最终流向：境内商家银行卡**

这是**最关键的侦察节点**：

| 商家类型 | 常见行业 | 商家解释 | 侦察突破口 |
|---------|---------|---------|-----------|
| **实体商家** | 建材/五金/批发 | 境外客户付款，货物出口 | 查进出口单据、报关记录 |
| **电商商家** | 网店经营 | 跨境电商收款 | 查电商平台后台流水 |
| **个人商家** | 场外换U | 朋友转账，换U赚差价 | 每U仅赚几分钱，需深挖上家 |

**商家"正当解释"的识别技巧**：

1. **话术特征**："境外客户通过地下钱庄转账"
2. **证据特征**：能提供供货合同、发货记录、物流单据
3. **资金特征**：金额与"正当生意"不匹配（如小超市收到百万转账）
4. **时间特征**：收款时间与受害人转账时间高度吻合

**侦察突破口（修正版）**：

| 突破口 | 具体方法 | 证据类型 | 侦察价值 |
|--------|---------|---------|---------|
| 洗钱车队 | 追踪操作员身份、资金归集点 | ⭐⭐⭐⭐⭐ 最高 |
| 卡主银行卡 | 调取转账记录、流水 | 银行记录 ✅ | 卡主是证人，可突破 |
| 境内商家 | 分析收款时间、金额匹配度 | 银行流水 ✅ | 深挖资金去向 |
| OTC记录 | 交易所KYC数据 | 交易所协查 ✅ | 追踪USDT来源 |
| 受害人时间线 | 串并分析多起案件 | 资金时间匹配 | 发现洗钱车队 |

#### 手法二：线下取现回U

```
┌─────────────────────────────────────────────────────────────────┐
│                       线下取现回U流程图                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   受害人 ──①取现金──→ 指定接头人（诈骗团伙成员）                  │
│                              │                                  │
│                              │ ──②现场买U──→ 诈骗团伙钱包         │
│                              ↓                                  │
│                        接头人处理：                               │
│                        现金交给上线 / 留下佣金                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**运作流程**：
1. 受害人按照诈骗团伙指令，到银行提取现金
2. 受害人与指定"接头人"见面，当面交付现金
3. 接头人现场操作手机，购买USDT并转账给诈骗团伙
4. 接头人通常获得1%-3%的佣金

**侦查突破口**：

| 突破口 | 具体方法 | 证据类型 |
|--------|----------|----------|
| 银行取款监控 | 调取银行ATM/柜台监控录像 | 视频证据 ✅ |
| 手机信令定位 | 追踪接头人手机信号位置 | 电子数据 ✅ |
| 现金交接地 | 调查交付地点周边监控 | 视频证据 ✅ |
| 资金链断裂 | 现金无法在链上追踪 | ⚠️ 盲区 |

**优势**：完全切断银行流水证据链
**劣势**：有线下监控和人证，接头人有被抓风险

**运作流程**：

2. 物品被直接交付或通过快递寄送到指定地址
3. 诈骗团伙成员（收货人）将实物变现
4. 通过OTC渠道购买USDT并转移

**侦查突破口**：

| 突破口 | 具体方法 | 证据类型 |
|--------|----------|----------|
| 店铺监控 | 调取购买时的店铺监控 | 视频证据 ✅ |
| 物流单号 | 追踪快递信息，定位收货人 | 物流数据 ✅ |
| 实物回收渠道 | 调查奢侈品回收店、金店 | 证人证言 ✅ |
| 二手交易平台 | 监控咸鱼、58同城等 | 平台数据 ✅ |
| 金额特殊 | 实物金额大（50万+），特征明显 | 分析研判 |

**优势**：资金量大，洗钱成本低（通过转卖差价获利）
**劣势**：受害风险最高（金条等实物交出后几乎无法追回）

## 1.2 公群担保流程详解

### 1.2.1 什么是公群担保？

**定义**：公群担保是一种在Telegram群组中进行的USDT交易担保服务，用于解决"先打钱还是先打货"的信任问题。

**为什么需要担保？**
- 在USDT交易中，双方互不信任
- 买家（诈盘口）怕打了USDT拿不到货（现金/实物）
- 卖家（U商）怕给了货收不到USDT
- 担保人作为第三方，持有USDT押金，确保交易顺利完成

---

### 1.2.2 汇旺担保流程详解（行业标准版）

> ⚠️ **重要修正**：本流程基于真实汇旺担保交易规则整理。与早期"受害人→担保人→U商→受害人"的错误描述不同，正确的流程是：**诈盘口（买家/资金方）→ 担保方 → U商（卖家/实物方）**。受害人不直接参与担保交易。

#### 角色定义

| 角色 | 真实名称 | 提供内容 | 接收内容 |
|------|----------|----------|----------|
| **买家/资金方** | 诈盘口/盘口 | USDT（资金） | 现金/实物/银行卡 |
| **卖家/实物方** | U商 | 现金/实物/银行卡 | USDT（资金） |
| **担保方** | 汇旺担保官方 | 资金托管服务 | 佣金（1%-3%） |
| **受害人** | 资金受害者 | 资金（被骗） | ❌ 不参与担保流程 |

## 1.4 境内商家配合模式识别专题（2024年新增）

### 1.4.1 商家配合的典型特征

**为什么商家会配合？**

| 商家类型 | 配合动机 | 常见话术 |
|---------|---------|---------|
| 主动参与 | 收取洗钱佣金（每笔1-3%） | "正常生意，不清楚资金来源" |
| 被迫配合 | 收到风控警告后妥协 | "我也没办法，他们威胁我" |
| 不知情 | 真的以为是正当交易 | "客户说是跨境付款" |

**识别商家配合的关键指标**：

1. **资金时间匹配度**
   - 商家收款时间与受害人转账时间差 ≤ 2小时 → ⭐⭐⭐⭐⭐ 高可疑
   - 商家收款时间与受害人转账时间差 2-24小时 → ⭐⭐⭐ 中可疑
   - 商家收款时间与受害人转账时间差 > 24小时 → 需进一步调查

2. **金额匹配分析**
   ```
   公式：匹配度 = 受害人损失金额 / 商家收款金额 × 100%
   
   匹配度 ≥ 80% → 高度可疑（可能是同一笔资金）
   匹配度 50-80% → 中度可疑（可能是资金分流）
   匹配度 < 50% → 需结合其他证据判断
   ```

3. **商家经营特征**
   | 异常特征 | 正常特征 | 可疑度 |
   |---------|---------|-------|
   | 小超市月流水百万 | 符合经营规模 | ⭐⭐⭐⭐⭐ |
   | 网店刚开就有跨境订单 | 正常电商运营周期 | ⭐⭐⭐⭐ |
   | 建材批发收到"货款" | 需核实进出口单据 | ⭐⭐⭐ |
   | 个人账户频繁收大额 | 正常个人账户 | ⭐⭐⭐⭐ |

### 1.4.2 "正当解释"的验证方法

**供货材料验证**：

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                            供货材料真实性验证流程                                 │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   Step 1: 收集供货材料                                                          │
│   ├── 供货合同（重点：签署时间、金额、货物类型）                                   │
│   ├── 发货记录（物流单号、收货地址）                                              │
│   └── 付款凭证（银行转账记录、收款收据）                                          │
│                    ↓                                                           │
│   Step 2: 交叉验证                                                              │
│   ├── 物流单号 → 物流公司官网查询真伪                                           │
│   ├── 合同金额 → 与税务申报比对                                                 │
│   └── 货物类型 → 与工商登记经营范围比对                                         │
│                    ↓                                                           │
│   Step 3: 时间线重建                                                            │
│   ├── 受害人转账时间                                                           │
│   ├── 商家收款时间                                                              │
│   ├── 声称的"发货时间"                                                         │
│   └── 时间矛盾 = 可疑                                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**"地下钱庄"线索追查**：

1. **资金链路追踪**
   - 追踪商家银行卡的资金来源（是否有"中转账户"）
   - 分析收款时间规律（是否与特定诈骗案件高度吻合）

2. **关联分析**
   - 同一商家是否涉及多起电诈案件
   - 商家银行卡是否与洗钱车队有关联

3. **实地调查**
   - 商家实际经营地址与注册地址是否一致
   - 经营规模是否与"月流水百万"匹配

### 1.4.3 场外换U识别

**少部分个人的解释模式**：

> "这是我朋友转给我的，说是让我帮忙换U，每U给他几分钱手续费"

**识别特征**：

| 特征 | 正常换U | 场外换U（可疑） |
|------|---------|----------------|
| 换U频率 | 偶尔一次 | 频繁、规律 |
| 单笔金额 | 大额整数 | 零散、有尾数 |
| 换U对象 | 熟人 | 不认识的人 |
| 佣金比例 | 正常市场价 | 每U仅几分钱 |

**侦察价值**：场外换U的"下家"可能成为洗钱车队的线索

## 2.1 地址身份指纹分析

### 2.1.1 什么是"地址指纹"？

**定义**：区块链上的地址本身是匿名的，但地址的交易行为会形成独特的"指纹"特征，通过分析这些特征可以推断地址的身份类型。

**核心思想**：
> 💡 同一个地址的多次交易会形成规律，这些规律就是地址的"指纹"。

---

### 2.1.2 五维指纹分析法

#### 维度一：交易频率

| 频率 | 日均笔数 | 身份推断 | 典型场景 |
|------|----------|----------|----------|
| 超高频 | >200笔/天 | 交易所量化/机器人 | 做市商、套利机器人 |
| 高频 | 50-200笔/天 | U商/OTC商家 | 专业USDT交易商 |
| 中频 | 10-50笔/天 | 中介/小U商 | 小规模OTC |
| 低频 | <10笔/天 | 普通用户 | 个人钱包 |
| 单次 | 1笔 | 一次性地址 | 赃款中转、一次性使用 |

**实操判断**：
```
工具：Etherscan → 点击地址 → 查看"Tansactions"标签
观察：交易时间分布图 + 交易总数
```

---

#### 维度二：交易对手数量

| 对手数量 | 特征 | 身份推断 |
|----------|------|----------|
| 1个固定对手 | 单一资金往来 | 熟人转账、个人钱包 |
| 2-10个对手 | 有限交易网络 | 小规模OTC、中介 |
| 10-50个对手 | 较广交易网络 | 中U商、归集地址 |
| 50+个对手 | 极广交易网络 | 大型U商、交易所热钱包 |

**实操判断**：
```
工具：Etherscan → "Token Transfers"标签
观察：Token转移的来源和去向地址数量
```

---

#### 维度三：金额模式

| 金额模式 | 特征 | 身份推断 |
|----------|------|----------|
| 整数模式 | 全部为1000、5000、10000等整数 | U商OTC（固定额度） |
| 零散模式 | 金额不整，有零头 | 普通用户 |
| 递增模式 | 金额逐笔增大 | 归集地址（小额→归集） |
| 递减模式 | 金额逐笔减小 | 分发地址（大额→分发） |
| 固定+零头 | 如1000.5、5000.3 | U商（含手续费） |

**实操判断**：
```
工具：Etherscan → "Internal Txns" 或 "Erc20 Token Txns"
观察：Token转移金额列表
```

---

#### 维度四：时间分布

| 时间分布 | 特征 | 身份推断 |
|----------|------|----------|
| 24h均匀分布 | 无休眠、无规律 | 机器人/自动化操作 |
| 白天活跃 | 北京时间9:00-18:00 | 中国U商 |
| 夜间活跃 | UTC时间16:00-02:00 | 海外U商 |
| 偶尔活跃 | 间隔数天 | 普通用户 |
| 节假日无休 | 全年无休 | 专业运营 |

**实操判断**：
```
工具：Etherscan → "Transactions" → 查看时间轴图
时区换算：北京时间 = UTC+8
```

---

#### 维度五：Gas设置（ETH链）

| Gas设置 | 特征 | 身份推断 |
|---------|------|----------|
| 固定Gas | 始终20/30/50 Gwei | 机器人/自动化脚本 |
| 浮动Gas | 根据网络状况调整 | 手动操作 |
| 超低Gas | 始终最低Gas | 成本敏感型U商 |
| 超高Gas | 始终最高Gas | 紧急转账/不在意成本 |

**实操判断**：
```

```

### 2.1.3 五维综合判断矩阵

```
├─────────────────────────────────────────────────────────────────────┤
```

## 3.1 跨链交易追踪

### 3.1.1 什么是跨链交易？

**定义**：跨链交易是指将资产从一条区块链转移到另一条区块链的过程。

**为什么诈骗团伙喜欢用跨链？**
1. **增加追踪难度** —— 不同链的数据分散，追查成本高
2. **混淆资金来源** —— 资金换链后，与原始来源的关联被切断
3. **绕过监控** —— 部分链的监管较弱

---

### 3.1.2 跨链交易识别三特征

当你在ETH链上追踪资金时，发现：
1. **对手地址是合约**（不是普通EOA地址）
2. **交易数据中带目标链地址参数**（如BSC地址）
3. **金额与费用结构特殊**（跨链费+手续费）

→ 这笔交易很可能是跨链交易！

---

### 3.1.3 主流跨链桥速查表

| 跨链桥 | 支持链 | 合约地址特征 | 查询入口 |
|--------|--------|-------------|----------|
| Multichain (AnySwap) | ETH/BSC/Polygon/Fantom等 | 固定前缀合约 | multiswap.org |
| Stargate | ETH/Arbitrum/Optimism/Metis | LayerZero协议 | layerzero.network |
| Across | ETH/Mainnet L2s | Hop协议 | across.to |
| SWFT | 30+主流链 | 流动性池模式 | swft.pro |
| Celer | 多链 | 流动性桥接 | cbridge.celer.network |
| Hyphen | ETH/BSC/Polygon | 即时跨链 | hyphen.biconomy.com |

---

### 3.1.4 跨链追踪三步法

```
┌─────────────────────────────────────────────────────────────────────┐
│                         跨链追踪三步法                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  第一步：定位跨链桥合约                                              │
│  ────────────────────                                               │
│  在源链找到调用跨链桥合约的交易                                       │
│  ↓                                                                  │
│  第二步：解析目标链信息                                              │
│  ────────────────────                                               │
│  解析交易Input Data，提取目标链接收地址                               │
│  ↓                                                                  │
│  第三步：切换目标链追踪                                              │
│  ────────────────────                                               │
│  在目标链浏览器查询接收地址，继续追踪                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**详细操作**：

**第一步：定位跨链桥合约**
```
1. 在Etherscan查看交易详情
2. 检查"To"字段是否为合约地址（Contract类型）
3. 对比已知跨链桥合约列表
4. 如果匹配，记录跨链桥名称
```

**第二步：解析Input Data**
```
1. 点击交易详情中的"Click to see More"
2. 展开"Input Data"部分
3. 寻找目标链地址（通常是另一个链的地址格式）
   - BSC地址：0x开头，42位
   - TRX地址：T开头，34位
   - SOL地址：sol开头
```

**第三步：目标链追踪**
```
1. 切换到目标链浏览器
2. 输入接收到的地址
3. 继续追踪资金流向
```

## 3.2 零知识证明与龙卷风Cash

### 3.2.1 什么是零知识证明？

**一句话解释**：
> 💡 你能证明你知道一件事，但不用说出这件事具体是什么。

**现实类比**：
```
传统证明：把钥匙给你看 → 你知道我有钥匙
零知识证明：把门打开 → 证明我有钥匙，但你没看到钥匙长什么样
```

**在区块链中的应用**：
- 证明我有这笔钱的所有权
- 但不暴露我的具体金额或地址
- 通过数学加密实现

---

### 3.2.2 龙卷风Cash运作机制

#### 存款过程

```
┌─────────────────────────────────────────────────────────────────┐
│                        Tornado Cash 存款                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户钱包 ──[转账ETH/USDT]──→ Tornado Cash 合约                  │
│       │                              │                          │
│       │                              │ 合约收款                  │
│       │                              │ 进入匿名池                │
│       │                              │ 与其他存款混合            │
│       ↓                              ↓                          │
│   生成 Note（存单凭证）                                          │
│                                                                 │
│   Note = 秘密值 + 零知识证明 + 存单编号                           │
│   特点：谁持有Note，谁就能取款                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 取款过程

```
┌─────────────────────────────────────────────────────────────────┐
│                        Tornado Cash 取款                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│       │                              │                          │
```

#### 龙卷风Cash的核心价值

**对于普通用户**：隐私保护
**对于犯罪分子**：资金混淆

### 3.2.3 NOTE详解

**NOTE是什么？**
```
NOTE = 存单凭证 = 秘密值 + 零知识证明 + 存单编号

格式示例：
{
  "currency": "ETH",
  "amount": 10,
  "note": "tornado-eth-10-0x...-0x...-0x..."
}
```

**NOTE的组成**：
| 组成部分 | 说明 | 示例 |
|----------|------|------|
| 币种 | 存入的Token类型 | ETH, USDT, DAI |
| 金额 | 存入的金额 | 0.1, 1, 10, 100 |
| 秘密值 | 加密的随机数 | 0x3f8e... |
| 存单编号 | 唯一的序列号 | n:0x1a2b... |
| 零知识证明 | 数学证明 | 加密数据 |

**NOTE在哪里找？**
1. 聊天记录（嫌疑人可能发给同伙）
2. 浏览器缓存（访问TC时本地存储）
3. TG消息（共享Note让别人代取）
4. 设备本地文件（备份的Note文件）

### 3.2.5 时间窗口匹配法

**为什么需要时间窗口分析？**

犯罪分子使用龙卷风Cash时：
- 存款时间：你知道的
- 取款时间：需要推断
- 金额匹配：可以帮助缩小范围

**时间窗口分析表**：

| 分析维度 | 具体方法 | 价值 |
|----------|----------|------|
| 72小时窗口 | 存款后72小时内为高频取款期 | ⭐⭐⭐⭐ 缩小范围 |
| 金额精确匹配 | 存款金额 = 取款金额 + Gas损耗 | ⭐⭐⭐⭐⭐ 关键证据 |
| Gas费指纹 | 固定Gas → 机器人/自动化 | ⭐⭐⭐ 辅助判断 |
| 交易对手关联 | 同一地址多次存/取TC | ⭐⭐⭐⭐ 行为画像 |

### 3.2.6 龙卷风Cash长期追踪策略

#### 策略一：交易对关联

**原理**：同一个地址多次使用TC → 形成行为模式

```
地址A的TC使用记录：
- 2024/3/1 存款 10 ETH → Note: n001
- 2024/3/15 取款 10 ETH → 取款地址: 0x...
- 2024/4/1 存款 10 ETH → Note: n002
- 2024/4/10 取款 10 ETH → 取款地址: 0x...

结论：该地址是职业TC使用者，可能为洗钱团伙
```

#### 策略二：金额匹配

**原理**：存款金额 = 取款金额 + Gas损耗

```
已知：存款10 ETH，Gas损耗约0.005 ETH
推断：取款金额约9.995 ETH
在取款列表中匹配金额相近的交易
```

#### 策略三：前后地址关联

**原理**：存款前的资金来源 vs 取款后的资金去向

```
存款前：地址A  ←←← 资金来源
    ↓
TC存款：10 ETH
    ↓
TC取款：10 ETH（去除Gas）
    ↓
取款后：地址B  →→→ 资金去向（通常是交易所/OFFRAMP）

通过追踪取款后的地址，可以找到资金出口
```

## 3.3 2025-2026年新手法速览

### 3.3.1 RAILGUN

**定位**：隐私池协议，隐私性更强

**与TC的区别**：
- Note机制类似，但隐私池更大

**侦查要点**：

### 3.3.2 Privacy Pools

**定位**：选择性披露机制

**特点**：
- 允许用户证明"我的存款在某个合法群体中"
- 但不暴露具体哪笔存款
- 新一代隐私协议

**侦查要点**：
- 关注异常的资金流动模式
- 结合传统追踪方法

### 3.3.3 TG Bot钱包洗钱

**代表工具**：
- Unibot（ETH链）
- Maestro（TG Bot）
- Banana Gun

**洗钱模式**：
```
诈骗资金 → Bot钱包 → 自动拆分成多笔普通ERC20转账 → 交易所
                                    ↑
                            伪装成正常交易
```

**侦查要点**：
- Bot钱包地址识别
- 交易金额和时间规律
- 短时间内的多笔分散转账

---

### 3.3.4 Pump.fun洗钱（Solana链）

**洗钱模式**：
```
诈骗资金 → 购买自己发行的Meme币 → 拉盘吸引跟风 → 砸盘变现
                            ↑
                    完全伪装成"投资亏损"
```

**特点**：
- 在Solana链上操作
- 资金流动伪装成正常投资
- 洗钱过程在链上看不出异常

**侦查要点**：
- 监控异常的新币交易
- 分析Meme币发行者地址
- 追踪资金最终去向
