## 五、教学内容

### 第1课时：电诈常见洗钱手法与公群担保流程

#### 1.1 三种常见回U手法（理论+实操 20分钟）

**⚠️ 2024年修正版：洗钱车队+商家配合模式**

**（1）卡接回U（洗钱车队版）** ⭐2024年主流
- **核心修正**：卡主仅提供银行卡，**洗钱车队操作员实际操作所有转账**
- 流程：受害人转账到银行卡 → 卡主仅提供卡 → **洗钱车队操作员实际操作** → 通过OTC买U → 资金转入**境内合法商家银行卡**
- 商家解释："境外客户通过地下钱庄转账"，能提供供货合同/发货记录
- 侦察突破口：
  - **洗钱车队操作员**是核心侦察对象（不是卡主）
  - 境内商家收款时间与受害人转账时间匹配度分析
  - 资金时间匹配公式：匹配度 ≥ 80% → 高度可疑

**（2）线下取现回U**
- 流程：受害人取现金 → 交给指定人 → 对方现场买U转走
- 侦查突破口：取款监控、手机信令定位、现金交接地排查
- 优势/劣势：切断银行流水证据链，但有监控和人证

**（3）线下取实物回U**
- 流程：受害人购买黄金/奢侈品等实物 → 交接收货 → 对方变现买U转走
- 侦查突破口：物流单号、店铺监控、实物回收渠道
- 优势/劣势：金额大（50万+），证据留存度最低

**洗钱车队内部结构（新增）**：
- 车队老板：指挥、分配任务、结算佣金
- 操作员：**实际操作转账，核心侦察对象**
- 商家配合：提供"正当交易"证明，解释资金来源

**三种手法对比总结表（学员课上填写）**
- 对比维度：核心角色、卡主角色、资金去向、侦察难度、洗钱成本
- 2024年新特征：卡接回U侦察难度最高（资金"正当化"）

#### 1.2 公群担保流程（理论+实操 15分钟）

**⚠️ 流程修正（基于汇旺担保真实规则）**：原"受害人→担保人→U商→受害人"的描述有误

**9节点标准流程（正确版）：**
```
①供需双方联系担保官方建群
②在交易群确认商品/价格/数量
③交易员汇总，双方最终确认
④买家（诈盘口）把USDT转入担保方上押账户  ← 关键节点：上押USDT
⑤担保方确认收款，通知卖家发货
⑥卖家（U商）发货，提供发货凭证
⑦买家确认收货
⑧担保方扣佣金，向卖家解押放款USDT  ← 关键节点：放款USDT
⑨卖家确认收款，交易完成
```

**角色定义（修正版）**：

| 角色 | 正确理解 | 提供内容 | 接收内容 |
|------|---------|---------|---------|
| 买家/资金方 | 诈盘口/盘口 | USDT | 现金/实物 |
| 卖家/实物方 | U商 | 现金/实物 | USDT |
| 担保方 | 汇旺担保官方 | 资金托管 | 佣金 |
| 受害人 | ❌ 不直接参与担保交易 |

**各节点侦察方向：**
- ④上押USDT：获取TxHash，定位诈盘口地址
- ⑧放款USDT：追踪放款去向，定位U商地址
- 关键侦察价值：追踪④到⑧的USDT流向即为担保交易证据链

**练习**：给2个真实改编案例，学员标注诈盘口、U商、担保方三方角色

#### 1.3 证据调取与保存（理论+实操 10分钟）

**调取常规方式：**
1. 链上截图（Etherscan/Tronscan/BSCScan）
2. 地址标签查询（OKLink等）
3. 浏览器查询（mempool.space等）
4. 交易所协查（调取证据通知书）

**保存常用方式：**
1. 截图命名规范：案件号_地址_TxHash_日期.png
2. TxHash记录表（Excel）：序号/TxHash/链类型/时间/金额/涉及地址/截图路径
3. 区块链存证（哈希固化）
4. 证据文件夹层级：案件编号/1.接警记录/2.链上数据/3.交易所数据/4.分析报告

### 第2课时：钱包地址特征分析与工具链实战

#### 2.1 交易行为身份指纹（理论+实操 10分钟）

**5维指纹判断标准：**
- 交易频率：高频(>50笔/天)→U商/机器人；低频→个人
- 对手数量：多对手→U商；单一对手→个人
- 金额模式：整数→U商；零散→个人；递增式→归集地址
- 时间分布：24h活跃→机器人；白天→个人
- Gas设置：固定→自动化操作；浮动→手动

**实操练习**：判断3个地址分别属于U商/个人/归集地址

#### 2.2 追踪工具链实操（理论+实操 25分钟）

**工具链总览（从简单到高级）：**
- 基础查询：Etherscan/Tronscan/BSCScan
- 地址追踪：**无匿(0xNazi)**、**重明(ChainEye)**
- 跨链穿透：**链币追**、**氪金道具**
- 消息爬取：**TG公群消息爬取软件**
- 渗透工具：**安垚**、**雷霆**、**白泽**

**实操内容：**
1. 无匿：输入地址→自动生成3层资金流向图→截图保存
2. 重明：对比与无匿的输出差异
3. 链币追：跨链交易识别→标注跨链桥节点
4. 氪金道具：混币器穿透功能演示
5. TG爬取：配置API→加入目标公群→关键词过滤→导出担保交易对话
6. 渗透工具：安垚查手机归属、雷霆查链路节点、白泽输出综合画像

**综合分析报告输出：地址画像报告模板**

### 第3课时：跨链交易追踪与龙卷风Cash

#### 3.1 跨链交易追踪方法（理论+实操 20分钟）

**跨链行为识别三特征：**
1. 对手地址是合约（跨链桥合约）
2. 交易数据中带目标链地址参数
3. 金额与费用结构特殊

**主流跨链桥速查：**
| 桥梁 | 合约地址特征 | 查询工具 |
|------|------------|---------|
| Multichain | 0x开头固定 | multiswap.org |
| Stargate | 固定桥合约 | layerzero.network |
| Across | 固定桥合约 | across.to |
| SWFT | 流动性池模式 | swft.pro |

**跨链追踪三步法：**
1. 源链找到跨链桥合约调用交易
2. 解析Input Data中的目标链接收地址
3. 在目标链浏览器查询该地址

**实操**：一笔ETH→BSC跨链交易全流程追踪

#### 3.2 零知识证明与龙卷风Cash（理论+实操 15分钟）

**零知识证明核心概念**：能证明你知道一件事，但不用说出这件事具体是什么

**龙卷风Cash运作机制：**
- 存款→生成Note（存单凭证，含秘密值）
- 等待→资金进入匿名池混合
- 取款→出示Note→证明有取款权→合约释放U→但"没告诉你从哪个存款地址取出"

**Note = 取款凭证 = 秘密值+零知识证明+存单编号**

**NOTE搜查方法论（核心实操）：**
- NOTE来源：聊天记录/浏览器缓存/本地文件/TG消息/交易所提现记录
- NOTE格式：{"currency":"ETH","amount":10,"note":"0x..."}
- 搜查步骤：提取NOTE信息→用存单编号查存款交易→确定时间窗口→关联取款交易→追踪取款地址

**龙卷风长期追踪策略：**
- 交易对关联、金额匹配、时间关联（存款后72小时取款高频窗口）、Gas费指纹、前后地址关联

#### 3.3 2025-2026手法更新速览（理论 5分钟）

- RAILGUN（隐私池替代品）
- Privacy Pools（选择性披露）
- TG Bot钱包洗钱（Unibot/Maestro）
- Pump.fun洗钱（Solana链meme币模式）

### 第4课时：经侦虚拟币平台实战——何文榕案带练

#### 4.1 平台界面速览（理论 2分钟）

三大功能区：交易查询区、分析面板区、报告输出区

#### 4.2 全程带练6步骤（实操 38分钟）

**Step 1：还原案件起点**
- 输入受害人TxHash→查看交易详情→标记受害人地址→扩展查询→截图命名保存
- **限时2分钟完成**

**Step 2：首层资金追踪**
- 追踪所有进账和出账→识别地址特征→绘制首层资金流向图→标注第一个可疑地址
- **注意**：第一层地址透露团伙的资金归集模式

**Step 3：多层穿透（3-5层）**
- 逐层追踪→遇跨链桥切换目标链→遇交易所标记类型→每两层截图一次
- **注意**：资金经过多层的损耗情况

**Step 4：工具链综合应用（自选）**
- 地址特征不明确→无匿/重明补充查询
- 遇U商地址→回溯U商身份
- 资金流分叉→链币追聚合流向图
- 发现TGID/手机号→安垚/雷霆查关联

**Step 5：报告输出（平台一键生成）**
- 案件基础信息、资金流向图、关键发现（最少3条）、侦查建议（最少2条）、截图附件清单

**Step 6：组间互审**
- 完整度检查、命名规范检查、证据支撑检查、建议可执行性检查

#### 4.3 复盘与结课（理论+实操 5分钟）

高频错误复盘 → 最佳报告展示 → 颁奖 → 课程核心要点五句话 → 持续学习资源卡发放
